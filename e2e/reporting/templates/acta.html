Datos utilizados

Los datos de entrada simulan a un cliente plausible: nombre y apellido reales, dirección en Caracas, email a un dominio sin servicio (example.com), teléfono móvil venezolano. Estrategia de robustez del OCR

El paso de lectura de documentos es uno de los más sensibles del flujo: una falla del proveedor de IA detendría la suscripción. Para minimizar este riesgo, el provider Gemini implementa tres capas de defensa que actúan automáticamente sin intervención del usuario:

Capa Comportamiento Activación
1. Pre-procesado de imagen sharp normaliza la imagen a 2000 px en su lado más largo, JPEG de calidad 90 % con submuestreo 4:4:4, normalización de contraste y realce de bordes. Garantiza que el modelo reciba un input legible. SIEMPRE
2. Reintentos transitorios Errores de red (5xx, timeout, rate-limit 429) o JSON inválido se reintentan hasta 2 veces sobre el mismo modelo con backoff exponencial (≈600 ms, 1.2 s, 2.4 s + jitter). AUTOMÁTICA
3. Cadena de modelos Si los reintentos del modelo principal fallan, o el JSON resultante carece de los campos críticos del documento (ej. identificacion en cédula, placa en certificado), el provider baja al siguiente modelo de la cadena: gemini-2.5-pro → gemini-2.5-flash → gemini-2.5-flash-lite. AUTOMÁTICA
Cobertura de tests: esta lógica está cubierta por la suite geminiProvider.test.js (11 casos, 92 % de cobertura del archivo) donde se mockea el SDK de Google para validar la cadena de fallback, los reintentos por error transitorio, la detección de campos críticos vacíos y el comportamiento ante document type mismatch.

6. Glosario y Servicios Externos

RCV Responsabilidad Civil de Vehículos. Póliza de seguro obligatoria para circular en Venezuela.
INMA Catálogo Internacional de Marcas y Modelos Automotrices. La Mundial expone la cascada año → marca → modelo → versión a través de su API.
VIN Vehicle Identification Number. Serial único de 17 caracteres del vehículo.
OCR Optical Character Recognition. En este proyecto se utiliza Google Gemini 2.5 Pro como motor principal para extraer datos de la cédula, licencia, certificado y RIF. El proveedor implementa una cadena de respaldo automática (Pro → Flash → Flash-Lite) y reintentos con backoff exponencial para garantizar lectura aún ante errores transitorios o cuotas momentáneas del modelo principal.
OTP One Time Password. Código de un solo uso que SyPago envía al cliente para confirmar pagos por débito.
Cotización Cálculo preliminar de la prima de la póliza. Endpoint: POST /getCotizacionAuto.
Emisión Generación definitiva de la póliza en La Mundial. Endpoint: POST /createEmissionAuto.
SUDEBAN Superintendencia de Bancos de Venezuela. Define los códigos de banco usados en pagos móviles.

Cómo reproducir esta corrida

  1. Clonar el repositorio y ejecutar npm install en la raíz, en server/ y en frontend/.
  2. Configurar variables de entorno copiando .env.example a .env.
  3. Levantar la app: npm run dev (frontend en puerto 5180, backend en 4000).
  4. Ejecutar la suite completa: npm run qa:full. Esto corre tests unitarios, E2E y genera el ACTA.
  5. El reporte HTML interactivo de Playwright queda en qa-evidence/playwright-report/ y este PDF en qa-evidence/ACTA-DE-PRUEBAS.pdf.
